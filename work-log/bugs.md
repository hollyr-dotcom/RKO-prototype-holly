# Bug List

> A living list of known bugs. Agents should work through this list top-to-bottom, following the workflow defined in `.cursor/rules/agent-workflow.mdc`.

## How to use this file

- **Adding a bug:** Append a new entry at the bottom of the list using the template below.
- **Picking up a bug:** Move it from `[ ]` to `[x]` only after the fix has been planned, implemented, and tested.
- **Priority:** Items at the top of the list are highest priority. Re-order as needed.

## Template

```markdown
### BUG-XXX: Short description
- **Status:** Open | In Progress | Fixed
- **Severity:** Critical | High | Medium | Low
- **Reported:** YYYY-MM-DD
- **Description:** What is happening vs what should happen.
- **Steps to reproduce:**
  1. …
- **Expected behaviour:** …
- **Actual behaviour:** …
- **Affected area:** e.g. component name, page, feature
```
