# Task List

> A living list of planned work items that advance the solution. Agents should work through this list following the workflow defined in `.cursor/rules/agent-workflow.mdc`.

## How to use this file

- **Adding a task:** Append a new entry at the bottom of the list using the template below.
- **Picking up a task:** Move it from `[ ]` to `[x]` only after the work has been planned, implemented, and tested.
- **Priority:** Items at the top of the list are highest priority. Re-order as needed.
- **Dependencies:** Note any tasks that must be completed first.

## Template

```markdown
### TASK-XXX: Short description
- **Status:** Open | In Progress | Complete
- **Priority:** Critical | High | Medium | Low
- **Created:** YYYY-MM-DD
- **Description:** What needs to be built or changed, and why.
- **Acceptance criteria:**
  - [ ] …
- **Dependencies:** None | TASK-XXX
- **Affected area:** e.g. component name, page, feature
```
